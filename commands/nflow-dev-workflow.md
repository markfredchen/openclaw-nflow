# /nflow-dev-workflow

## 开发循环详细执行流程

**Phase:** Phase 8

**执行时机:** `/nflow-dev` 激活后

---

## 通知机制

每个节点完成后自动发送 Telegram 通知，包含：
- 节点名称和执行状态
- 任务详情
- 截图（如有）

**通知脚本用法：**
```bash
python3 scripts/nflow_notify.py \
    --node "节点名称" \
    --status success \
    --message "详情" \
    --screenshot "path/to/screenshot.png" \
    --story-id STORY-001 \
    --sprint sprint-01
```

## 完整执行流程

```
┌─────────────────────────────────────────────────────────────┐
│  确定执行哪个 Sprint                                        │
│  → 找到最新有未完成 Stories 的 Sprint                       │
│  → 发送通知: 节点开始                                      │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  检查 Sprint Stories 状态                                  │
│  user-stories-tracker.md                                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
                    还有未完成的 Stories?
                    ├── 是 → 继续执行
                    └── 否 → Sprint 完成 → 找下一个 Sprint
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  从 TODO Stories 中选择下一个 Story                         │
│  按优先级顺序（考虑依赖关系）                               │
│  → 发送通知: Story 已选择                                  │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  8.3 创建 Git Worktree                                    │
│  Git Workflow Master Agent                                │
│  → 发送通知: Worktree 创建结果                             │
└─────────────────────────────────────────────────────────────┘
                              ↓
        ┌──────────────────────┐  ┌──────────────────────┐
        │ 8.4 E2E 用例编写   │  │ 8.5 TDD 开发         │
        │ QA Agent            │  │ Developer Agent       │
        │ (并行)              │  │ (并行)                │
        │                    │  │                        │
        │ • 理解 Story       │  │ RED → GREEN → REFACTOR│
        │ • 编写 E2E 用例    │  │                        │
        │   (含截图步骤)     │  │                        │
        │ • 更新 stories.md   │  │                        │
        │ → 发送通知:        │  │ → 发送通知:           │
        │   E2E 用例编写完成 │  │   TDD 开发完成        │
        │   + 截图           │  │   + 截图（如有）      │
        └──────────────────────┘  └──────────────────────┘
                              ↓
                    两个任务都完成后
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  8.6 代码审查                                             │
│  Code Reviewer Agent                                      │
│  🔴 Blocker / 🟡 Suggestion / 💭 Nit                   │
│  → 发送通知: 审查结果                                     │
│  → 如有截图（审查问题），一并发送                          │
└─────────────────────────────────────────────────────────────┘
                              ↓
                    有 Blocker?
                    ├── 是 → 记录审查失败次数
                    │         ↓
                    │    审查失败次数 ≥ 3?
                    │    ├── 是 → 🤖 人工干预！发送紧急通知
                    │    └── 否 → 8.7 修复 → 重新审查
                    └── 否 → 继续
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  8.8 执行测试                                             │
│  QA Agent                                                │
│  → 发送通知: 测试执行结果                                 │
│  → 如有失败截图，一并发送                                 │
└─────────────────────────────────────────────────────────────┘
                              ↓
                    测试通过?
                    ├── 是 → 继续 8.9 E2E 测试
                    └── 否 → 测试失败次数 +1
                              ↓
                    测试失败次数 ≥ 3?
                    ├── 是 → 🤖 人工干预！发送紧急通知
                    └── 否 → 回退到代码编写
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  8.9 E2E 测试（如需要）                                  │
│  QA Agent                                                │
│  → 发送通知: E2E 测试结果                                 │
│  → 发送截图: 测试报告截图                                 │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  8.10 代码合并                                           │
│  Git Workflow Master Agent                                │
│  → 发送通知: 合并结果                                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  8.11 更新 Sprint 追踪                                   │
│  Sprint Prioritizer Agent                                │
│  → user-stories-tracker.md                              │
│  → 发送通知: Story 状态更新                               │
└─────────────────────────────────────────────────────────────┘
                              ↓
                         返回检查 Stories 状态
```

## 节点通知汇总

| 节点 | 状态 | 截图 | 说明 |
|------|------|------|------|
| Story 选择 | ✅/❌ | 否 | 选择结果 |
| Git Worktree | ✅/❌ | 否 | 创建结果 |
| E2E 用例编写 | ✅/❌ | 是 | 用例截图 |
| TDD 开发 | ✅/❌ | 可选 | 失败截图 |
| 代码审查 | ✅/❌ | 可选 | 问题截图 |
| 测试执行 | ✅/❌ | 是 | 失败截图 |
| E2E 测试 | ✅/❌ | 是 | 报告截图 |
| 代码合并 | ✅/❌ | 否 | 合并状态 |
| Tracker 更新 | ✅ | 否 | 状态确认 |
