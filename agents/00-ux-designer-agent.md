## Workflow

### Phase 2: UX Design

**前置输入:** `prd.md` + `architecture.md`

**执行步骤:**

```
1. 理解需求
   - 阅读 PRD 中的 User Story Map
   - 阅读 Architecture 中的前端架构

2. 识别页面清单
   - 根据 User Stories 提取所有需要的页面
   - 标注页面优先级（P0 / P1 / P2）

3. 设计每个页面的线框图
   - 使用 ASCII Art
   - 每个页面单独一个 .md 文件
   - 包含：页面标题、功能说明、ASCII 线框图、交互说明

4. 设计用户流程
   - 页面之间的跳转关系
   - 异常流程（空状态、错误状态、加载状态）

5. 评审和确认
   - 展示给用户确认
   - 根据反馈调整
```

## ASCII Wireframe 规范

### 元素符号库

```
┌─────────────────────────────┐  ← 容器/卡片
│                             │
│  ┌─────────┐  ┌─────────┐  │  ← 子模块
│  │  按钮   │  │  输入框  │  │
│  └─────────┘  └─────────┘  │
│                             │
│  [ 导航栏 ]  [ 搜索 ]  [头像]│  ← 顶部栏
│                             │
│  ┌─────────────────────────┐│
│  │      图文卡片            ││  ← 内容卡片
│  │  [图片]  标题   描述     ││
│  └─────────────────────────┘│
│                             │
│  ← 上一页    1/5    下一页 → │  ← 分页
│                             │
│  ─────────────────────────  │  ← 分隔线
│  (页脚信息)                  │
└─────────────────────────────┘
```

### 状态标注

```
[ 按钮 ]           ← 默认状态
[ 按钮 ] ⚠️        ← 警告/错误
[ 按钮 ] ✓         ← 成功状态
[ 按钮 ] ⏳        ← 加载状态
[ 按钮 ] 🔒        ← 禁用状态
```

### 交互说明格式

```markdown
## 交互说明

| 元素 | 交互 | 行为 |
|------|------|------|
| [登录按钮] | 点击 | 验证表单 → 调用 /api/auth/login |
| [头像] | 点击 | 弹出下拉菜单 |
| [列表项] | 左滑 | 显示删除按钮 |
```

## Wireframe Quality Checklist

```
□ 布局是否简洁，信息优先级清晰？
□ 操作路径是否最短？
□ 异常状态（空/错/加载）是否处理？
□ 关键元素是否有视觉引导？
□ 是否符合平台设计规范？
□ ASCII 线框图用等宽字体对齐？
```
